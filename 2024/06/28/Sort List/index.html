148. Sort List

給與一個單向 linked list，將其從小到大排序。

先回想一下 merge sort 的作法，先將陣列不斷對半切，直到所有陣列都剩下一個元素，再將它們比大小後合併，也就是 divide and conquer。

我們學過如何使用 fast-slow 找中點，我們也學過如何從小到大 merge list，組合起來就可以實現 merge sort。

複雜度

Merge Sort 的遞迴呼叫可以畫成一顆Binary Tree，這可以更好的分析時間與空間複雜度。

時間
在merge sort中，split不斷產生新問題，經過logN次分割後達到最小問題，接著做merge，最糟情況兩個陣列長度相同，每個元素都需要比較一次，故最多比較N次。我們做了logN次分割，也需要logN次合併，所以時間複雜度O(NlogN)。

空間
對 Linked List 分割時只做指標操作，但 recursive 需要空間存放 call stack，遞迴深度等於樹高，也就是 logN，所以用在 linked list 時空間複雜度 O(log n)。
